## Decorator (wrapper)
Структурный паттерн, динамически добавляющий объекту новые обязанности.
По смыслу, данный паттерн "заменяет" собой статическое расширение класса наследованием, которое должно расширять функционал базового класса, не меняя поведение основного.

## Применимость
- Динамическое добавление обязанностей объектам;
- Реализация обязанностей, которые могут быть сняты с объекта;
- Когда наследование по какой-либо причине не удобно или не применимо.
